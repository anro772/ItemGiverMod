THUNDERSTORE PACKAGING INSTRUCTIONS - ItemGiver v2.0.1
==============================================================================

STEP 1: CREATE THE ICON (256x256 PNG)
--------------------------------------
You need to create an icon.png file (exactly 256x256 pixels).

Quick options:
A) Use an online tool:
   - Visit: https://www.photopea.com/ (free Photoshop alternative)
   - Create new image: 256x256 pixels
   - Add text/graphics for your mod
   - Export as PNG -> Save as "icon.png"

B) Use GIMP/Photoshop:
   - Create 256x256 canvas
   - Design your icon
   - Export as "icon.png"

C) Use a placeholder:
   - Create simple colored square 256x256
   - Add "ItemGiver" text in center
   - Save as icon.png

SAVE IT TO: C:\Users\stefa\Desktop\dnSpy-net-win64\project\ItemGiverMod\ThunderstorePackage\icon.png


STEP 2: VERIFY PACKAGE CONTENTS
---------------------------------
Your ThunderstorePackage folder should contain:
✓ manifest.json           (Done - defines mod info and dependencies)
✓ README.md              (Done - displays on Thunderstore page)
✓ ItemGiver.dll          (Done - the actual mod)
✗ icon.png               (YOU NEED TO CREATE THIS - 256x256)


STEP 3: UPDATE manifest.json (OPTIONAL)
----------------------------------------
If you want to change the GitHub URL or description:
1. Open manifest.json
2. Edit the "website_url" field
3. Edit the "description" field (max 250 chars)
4. Save the file


STEP 4: CREATE THE ZIP FILE
----------------------------
IMPORTANT: The zip must contain the files DIRECTLY, not in a subfolder!

Correct structure:
  ItemGiver.zip
    ├── manifest.json
    ├── README.md
    ├── icon.png
    └── ItemGiver.dll

WRONG structure (do NOT do this):
  ItemGiver.zip
    └── ThunderstorePackage/
          ├── manifest.json
          ├── README.md
          ├── icon.png
          └── ItemGiver.dll

How to create the zip correctly:
1. Open the ThunderstorePackage folder
2. Select all 4 files (manifest.json, README.md, icon.png, ItemGiver.dll)
3. Right-click -> Send to -> Compressed (zipped) folder
4. Name it: ItemGiver.zip


STEP 5: UPLOAD TO THUNDERSTORE
-------------------------------
1. Go to: https://thunderstore.io/c/megabonk/create/
2. Log in with your account
3. Upload ItemGiver.zip
4. Thunderstore will validate the package
5. If everything is correct, your mod will be published!


WHAT HAPPENS FOR USERS:
------------------------
When someone installs your mod via mod manager (r2modman/Thunderstore Mod Manager):

1. Mod manager reads manifest.json
2. Sees dependency: "BepInEx-BepInExPack_IL2CPP-6.0.738"
3. Automatically downloads and installs BepInEx first
4. Then installs your ItemGiver.dll
5. User launches game - everything works!

Manual installation users will see the README.md with manual instructions.


UPDATING THE MOD:
-----------------
When you make changes:
1. Update VERSION in Plugin.cs (e.g., "2.0.2")
2. Update version in ItemGiverMod.csproj
3. Update version_number in manifest.json
4. Rebuild the DLL
5. Copy new DLL to ThunderstorePackage/
6. Update changelog in README.md
7. Create new zip file
8. Upload to Thunderstore


TROUBLESHOOTING:
----------------
Q: Upload fails - "Invalid package structure"
A: Make sure files are in root of zip, not in a subfolder

Q: Upload fails - "Icon resolution incorrect"
A: Icon must be EXACTLY 256x256 pixels PNG

Q: Upload fails - "Invalid manifest"
A: Check manifest.json syntax - must be valid JSON

Q: Users report "missing BepInEx"
A: They're installing manually - tell them to install BepInEx first
